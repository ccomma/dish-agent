# Environment Health Check & Pre-flight Verification Skill

在开始任何工作之前，自动运行完整的环境检查，确保所有依赖、凭证、构建环境都处于可用状态。如发现问题，自动尝试修复或创建 Agent 处理。

## 触发条件

- 开始任何新的工作会话
- 添加新依赖之前
- 运行构建失败后
- 任何怀疑环境问题时

## 检查项目（按顺序执行）

### 1. Maven 仓库连接检查
```bash
mvn dependency:resolve -s settings-test.xml
```
**成功标准**：BUILD SUCCESS，所有依赖解析成功

### 2. 凭证配置检查
检查以下配置是否存在且有效：
- `config.properties` 中的 `MINIMAX_API_KEY`
- `config.properties` 中的 `COHERE_API_KEY`（如使用 Reranking）

### 3. 依赖完整性检查
验证以下依赖能否正常解析：
- langchain4j (core)
- langchain4j-open-ai
- langchain4j-ollama
- langchain4j-milvus（如使用）
- langchain4j-cohere（如使用）

### 4. 编译验证
```bash
mvn compile -s settings-test.xml
```
**成功标准**：BUILD SUCCESS，无编译错误

### 5. API 版本兼容性检查
验证 pom.xml 中的依赖版本与代码中使用的 API 匹配

## 自动修复策略

| 问题类型 | 修复尝试 |
|---------|---------|
| 仓库连接超时 | 切换 settings-test.xml 镜像配置 |
| 依赖解析失败 | 检查版本号是否正确，尝试查找可用版本 |
| 编译错误 | 创建 Agent 分析错误并修复 |
| API 不兼容 | 创建 Agent 探索正确 API 并修复 |

## Agent 处理流程

当检查发现问题且无法自动修复时：

1. **创建分析 Agent**：
   - 分析错误原因
   - 确定修复方案
   - 执行修复

2. **验证 Agent**：
   - 运行修复后的检查
   - 确保问题已解决

3. **报告 Agent**：
   - 汇总检查结果
   - 报告已修复/未修复的问题

## 执行流程

1. 运行所有检查项
2. 收集检查结果
3. 如有问题，尝试自动修复
4. 如无法自动修复，创建 Agent 处理
5. 循环验证直到环境健康或报告无法解决的问题
6. 只有环境健康时才继续后续工作
